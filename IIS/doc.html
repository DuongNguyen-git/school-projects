Implementace

Každý uživatel má přístupné tlačítko Upravit profil, kde si může změnit některé osobní údaje a změnit heslo. Odhlásit se je možné přes tlačítko Odhlásit se případně dojde k automatickému odhlášení po určité době nečinnosti.

Došlo také k pár odchylkám, co se týče Use-Case diagramu navrhnutému v předmětu IDS, ředitel může kromě mazání účtů zaměstanců také přidávat či odebírat oddělení nebo si nechat vypsat vyšetření, které proběhly v nemocnici.

U doktora nám přišlo zbytečné dávat přidávat pacienta, čímž se nemusí zdržovat, protože to udělá sestra a pouze si nechá vypsat údaje o něm, k lékům má omezenější přístup, ale může si všechny léky projít.

U lékárníka přibyla položka vydat lék, což určitě je v praxi potřeba.

Zdravotní sestra také může zadat vyšetření a také zadat pacienta do systému, případně upravit jeho osobní informace pro ušetření času doktora.

Úvodní stránku načteme na index.php

Ve složce lib najdeme:

  • db-connect.php - třída pro připojení k databázi a s pomocnými funkcemi, které urychlí zadávání dotazů při psaní stránek
  • nemocnice.css - soubor se styly, který definuje jak budou stránky vypadat

Hlavní soubory

  • reditel.php - administrátorské rozhraní pro ředitele nemocnice
  • doktor.php - rozhraní pro doktory
  • sestra.php - rozhraní pro zdravotní sestry
  • lekarnik.php - administrátorské rozhraní pro lékárníky

Pomocné soubory

  • login.php - kotrola správného přihlášení
  • logout.php - odhlášení
  • search.php - slouží k úpravě osob
  • edit.php - slouží k uprávě profilu
  • uvod.php - obsahuje html hlavičku včetně importu některých javascriptů a kaskádových stylů
  • uvod2.php - obsahuje alternativní html hlavičku pro vyskakovací okna
  • zaver.php - uzavírá html soubor
  • db.sql - soubor pro vytvoření databáze a naplnění jí daty

Instalace

Nejprve je potřeba zkopírovat všechny soubory na server, poté naimportovat sql soubor do MySQL databáze.

Softwarové požadavky

  • MySQL
  • PHP 5.6.
